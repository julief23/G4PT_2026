AGAPE User Guide

1. Introduction

AGAPE (computAtional G-quadruplex Affinity PrEdiction) is a novel machine learning (ML)-based model designed to predict the binding and stabilizing potential of small molecules targeting G-quadruplexes (G4s). G4s, prevalent in telomeres and oncogene promoters, are promising therapeutic targets, but designing selective binders remains challenging.

Built upon a curated dataset of 1217 compounds annotated through Förster Resonance Energy Transfer (FRET) melting assays, AGAPE integrates 5666 molecular descriptors, both classical and quantum chemical, to capture features relevant to G4 recognition, driving researchers to predict the potential G4-stabilization of small molecules, including both organic ligands and metal complexes.

Among the trained ML models, XGBoost achieved the best performance with an accuracy of nearly 90%. SHAP analysis highlighted descriptors related to molecular topology, polarizability, and electrostatic potential as key contributors to classification.

AGAPE is deployed through a user-friendly web interface supporting batch prediction and secure data handling. This work provides a robust and interpretable tool for accelerating the discovery of selective G4-stabilizing compounds and highlights the value of integrating quantum chemical insights into ML-driven chemoinformatics.

The platform integrates molecular input handling, automatic Mordred descriptor computation, feature alignment, and machine learning prediction.

Step 1 — Enter a Job Name

• Provide a descriptive name for your prediction run.

Step 2 — Select a Model
  • • Lightweight Model — 75 selected features
  • • General Model — Currently locked
Step 3 — Choose Input Mode
AGAPE supports three molecular input methods.
  • • CSV Upload (Batch Mode) Upload up to 150 molecules in a single file.
  • • Single SMILES Input Enter one valid SMILES string.
  • • Molecule Editor Draw your structure using the integrated editor.
3. CSV Format Requirements
Your CSV must follow one of the supported formats.
Option A — SMILES Only

The file must contain a column named:

SMILES

AGAPE will automatically compute Mordred descriptors.

Option B — SMILES + Precomputed Descriptors

If descriptors are already computed, column names must match the trained model exactly. Missing features will trigger an error.

4. Internal Processing Workflow
  1. • SMILES validation
  2. • Canonicalization
  3. • Descriptor computation (if required)
  4. • Feature alignment
  5. • Model prediction
Invalid molecules are automatically filtered and reported.
5. Understanding the Results
Prediction Label
  • 0 — Inactive
  • 1 — Active
Probability Score

The probability ranges from 0 to 1.

  • Close to 1 → strong confidence in Active
  • Close to 0 → strong confidence in Inactive
  • Around 0.5 → uncertain prediction
A probability of 0.12 does not indicate a weak model. It indicates strong confidence that the compound is inactive.
6. Downloading Results

After prediction, results can be exported using the "Download Results (CSV)" button.

The exported file contains SMILES, prediction label, and probability score.
