Frequently Asked Questions

Find answers to common questions about using the G4PT molecular generation platform.

📘 Looking for detailed instructions? Access the G4PT User Guide

G4PT is an AI-assisted platform for generating and prioritizing novel G-quadruplex ligands. It combines natural language interpretation, MolGPT-based molecular generation, RDKit validation, and AGAPE-based ranking to return candidate molecules for further analysis.

AGAPE is mainly a predictive platform: it estimates whether a molecule is likely to stabilize G4 structures. G4PT focuses on molecular generation. AGAPE is used internally as a ranking module to prioritize generated candidates.

G4PT supports four generation modes:
  • De novo generation — generate novel G4-like ligands without a reference molecule.
  • Similar ligand generation — generate analogues of a known ligand.
  • Scaffold extension — grow a scaffold into a complete candidate molecule.
  • Scaffold hopping — generate related molecules with alternative scaffolds.

Use structured input when you already know the generation mode and reference molecule. Use the G4PT Assistant when you prefer to describe your request in natural language. The two input strategies are kept separate to avoid contradictory instructions.

Depending on the selected mode, G4PT can use:
  • No reference for de novo generation.
  • Ligand name, resolved into a molecular structure when possible.
  • SMILES, validated and canonicalized with RDKit.
  • Molecular editor input, allowing users to draw a structure directly.
Only one reference input method should be used at a time.

Qwen interprets natural language requests and converts them into structured molecular constraints. It does not replace MolGPT and it does not rank molecules. Its role is to help translate user instructions into parameters that the generation workflow can use.

MolGPT is the molecular generator used by G4PT. A single multitask MolGPT model supports de novo generation, analogue generation, scaffold extension, and scaffold hopping through instruction-style mode tags.

Generated molecules are validated using RDKit. Invalid structures are removed, valid molecules are canonicalized, duplicates are filtered, and molecular descriptors are computed before AGAPE-based prioritization.

G4PT returns ranked candidate molecules with information such as structure, SMILES, AGAPE score, molecular properties, synthetic accessibility estimates, similarity when relevant, and generation mode. Results can be exported as a CSV file.
