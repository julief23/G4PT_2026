G4PT User Guide

1. Introduction

G4PT is an AI-assisted platform designed to generate and prioritize novel G-quadruplex (G4) ligand candidates.

Unlike AGAPE, which focuses on predicting G4 ligand activity, G4PT focuses on molecular generation. AGAPE is used internally as a ranking module to help prioritize generated molecules.

The platform combines natural language interpretation, MolGPT-based molecular generation, RDKit validation and filtering, descriptor calculation, AGAPE prediction, ranking, and CSV export.

2. Running a Generation Job
Step 1 — Enter a Job Name

Provide a short descriptive name for your generation run. This helps identify the output files and results.

Step 2 — Choose an Input Strategy

G4PT provides two mutually exclusive input strategies:

  • Structured input — manually select the generation mode and provide the required reference input.
  • G4PT Assistant — describe your request in natural language. Qwen converts the request into structured constraints.
Only one input strategy can be used at a time to avoid contradictory instructions.
3. Structured Input
Generation Modes
  • De Novo Generation — generate novel G4-like ligands without a reference molecule.
  • Similar Ligand Generation — generate analogues of a known ligand.
  • Scaffold Extension — grow a scaffold into a complete candidate molecule.
  • Scaffold Hopping — generate chemically related molecules with alternative scaffolds.
Reference Input

Depending on the selected generation mode, the user may provide one reference input.

  • No reference — recommended for de novo generation.
  • Ligand name — resolved into a molecular structure when possible.
  • SMILES — validated and canonicalized before generation.
  • Molecular editor — draw a structure directly in the browser.
Use only one reference input method at a time: ligand name, SMILES, or molecular editor.
4. G4PT Assistant

The G4PT Assistant uses Qwen to interpret natural language requests and extract structured molecular constraints.

Example instructions:

  • Generate lighter analogues of pyridostatin.
  • Generate diverse G4-like ligands with improved synthetic accessibility.
  • Extend this scaffold into complete candidate molecules.
  • Generate scaffold-hopped analogues of BRACO-19.
Qwen does not generate molecules directly. It prepares structured instructions that guide the MolGPT generation workflow.
5. Internal Workflow

After submission, G4PT performs the following automated steps:

  1. Input interpretation or structured input parsing
  2. Ligand name, SMILES, or scaffold resolution
  3. SMILES validation and canonicalization using RDKit
  4. Molecular generation using MolGPT
  5. RDKit validation and duplicate removal
  6. Descriptor calculation
  7. AGAPE prediction
  8. Ranking of generated candidates
  9. Results display and CSV export
Invalid or unparsable generated molecules are automatically removed before ranking.
6. Results

G4PT displays ranked generated molecules and associated molecular information.

  • Rank — candidate position after filtering and prioritization.
  • Structure — molecular structure preview.
  • SMILES — canonical molecular representation.
  • AGAPE score — predicted G4 activity score.
  • MW — molecular weight.
  • LogP — estimated lipophilicity.
  • SA score — synthetic accessibility estimate.
  • Similarity — similarity to the reference molecule when relevant.
  • Generation mode — mode used to generate the candidate.
Generated molecules are computational suggestions. They should be validated with additional computational and experimental methods.
7. Exporting Results

After generation and ranking, results can be downloaded as a CSV file.

The exported file contains generated molecules, scores, molecular properties, generation mode, and relevant filtering information.
